# Описание проекта по пунктам

## 1. Название
**STOM Dental Bot** — Telegram-ассистент стоматологической клиники «STOM» (г. Гавар, Армения).

## 2. Проблема
Пациенты клиники ежедневно задают однотипные вопросы: какие услуги есть, сколько стоит лечение, как подготовиться к приёму, что делать после процедуры, как записаться. Администраторы и врачи тратят на это рабочее время, а в нерабочие часы или при высокой нагрузке пациенты остаются без ответа. Проект решает задачу автоматизации первой линии коммуникации, повышая доступность клиники круглосуточно и разгружая персонал.

## 3. Пользователь
- **Основная аудитория:** пациенты и потенциальные клиенты клиники STOM (взрослые и дети), жители г. Гавар и близлежащих районов Армении.
- **Косвенные пользователи:** администрация и медицинский персонал клиники, которым бот передаёт уже отфильтрованные и конкретные обращения.

## 4. Основная функция
Что подаётся на вход:
- Текстовые сообщения и команды пользователя в Telegram (/start, /help, /prices, /contacts, /calendar, а также произвольные вопросы).
- Системный промпт (файл `stdn.txt`) + база знаний клиники в Markdown (`knowledge/FAQ.md`, `knowledge/prices.md`, `knowledge/postop.md`).
- История диалога пользователя (до 10 последних сообщений).

Что получается на выходе:
- Естественно-языковые ответы, сгенерированные LLM на основе загруженной базы знаний и системного промпта.
- Прямые ответы по командам: приветствие, прайс-лист, контакты клиники, inline-кнопка для онлайн-записи через Google Calendar.

Задействованные инструменты:
- Telegram Bot API (python-telegram-bot ≥20).
- LLM-провайдеры: **Ollama** (локально, по умолчанию), **Anthropic Claude**, **OpenAI**.
- Файловая система для хранения базы знаний, конфигурации и логов.

## 5. Формат результата
Пользователь получает **текстовые сообщения в Telegram**:
- Ответы на вопросы в свободной форме (Markdown-разметка).
- Прайс-лист, разбитый на сообщения (до 4000 символов).
- Inline-кнопку со ссылкой на онлайн-запись.
- Контактную информацию клиники.
- Системные уведомления при ошибках или недоступности LLM.

## 6. Минимальный функционал (MVP)
Реализовано:
- Обработка команд `/start`, `/help`, `/prices`, `/contacts`, `/calendar`.
- Обработка произвольных текстовых вопросов через LLM с учётом системного промпта и базы знаний.
- История диалога в памяти (ограничение: 10 сообщений).
- Поддержка трёх LLM-провайдеров (переключаются через `.env`).
- Логирование работы бота в файл `logs/bot.log`.
- Запуск и остановка через shell-скрипты (`start_bot.sh`, `stop_bot.sh`).

Планируется в будущем:
- Интеграция с реальным Google Calendar API (сейчас только ссылка).
- Персистентное хранение истории диалогов (SQLite или другая БД).
- Автоматические напоминания о приёме (SMS/ Telegram push).
- Панель администратора для редактирования базы знаний без перезапуска бота.

## 7. Технологии
- **Язык программирования:** Python 3.
- **Основные библиотеки:**
  - `python-telegram-bot` (взаимодействие с Telegram API).
  - `python-dotenv` (загрузка переменных окружения из `.env`).
  - `requests` (HTTP-запросы к локальному Ollama).
  - `anthropic` (официальный SDK для Claude).
  - `openai` (официальный SDK для OpenAI API).
- **Хранение данных:** файловая система (Markdown-файлы для знаний, `.env` для конфигурации, текстовые логи).
- **Форматы ответов:** Markdown для Telegram-сообщений, JSON для API Ollama, текстовый формат для логов.
- **Архитектура:** монолитное Python-приложение, единый entry-point (`bot.py`), polling-режим.
