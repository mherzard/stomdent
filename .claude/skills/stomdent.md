# STOM Dental Bot — навык работы с репозиторием

## Общая информация

- **Название**: STOM Dental Bot (MVP)
- **Язык**: Python 3
- **Платформа**: Telegram (python-telegram-bot >=20)
- **LLM**: Поддержка Anthropic, OpenAI, Ollama
- **Клиника**: STOM, г. Гавар, Армения
- **Рабочая директория**: `/Users/macbook/myai/stomdent`

## Структура проекта

```
/Users/macbook/myai/stomdent/
├── bot.py                 # Главный файл бота (весь код в одном файле)
├── stdn.txt               # Системный промпт (стиль, данные клиники)
├── requirements.txt       # Python-зависимости
├── .env.example           # Шаблон переменных окружения
├── .env                   # Реальные ключи (НЕ коммитить!)
├── start_bot.sh           # Запуск в фоне (nohup + PID)
├── stop_bot.sh            # Остановка по PID
├── logs/                  # Логи работы
│   └── bot.log
├── knowledge/             # База знаний (Markdown)
│   ├── FAQ.md             # 50 вопросов пациентов
│   ├── prices.md          # Прайс-лист в AMD
│   └── postop.md          # Послепроцедурные рекомендации
└── venv/                  # Виртуальное окружение
```

## Ключевые правила при работе

1. **Все команды выполняйте из `/Users/macbook/myai/stomdent`**.
2. **Никогда не коммитьте `.env`** — там хранятся API-ключи.
3. **База знаний — единственный источник правды** для бота. Любые изменения в ценах, услугах или рекомендациях вносятся только в `knowledge/*.md`.
4. **После изменения `knowledge/` или `stdn.txt` бот нужно перезапустить** — он читает файлы при старте.
5. **Код бота в одном файле** (`bot.py`). Держите его так же читаемым — секции разделены комментариями `-----------`.
6. **README.md должен всегда соответствовать реальности** — при изменении структуры, команд или зависимостей обновляйте и его.

## Запуск, остановка и проверка

### Запуск
```bash
cd /Users/macbook/myai/stomdent
./start_bot.sh
```
Скрипт автоматически создаст `venv`, установит зависимости и запустит бота в фоне. PID сохранится в `bot.pid`.

### Проверка статуса
```bash
ps -p $(cat bot.pid) 2>/dev/null && echo "Бот работает" || echo "Бот не запущен"
```

### Логи в реальном времени
```bash
tail -f /Users/macbook/myai/stomdent/logs/bot.log
```

### Остановка
```bash
cd /Users/macbook/myai/stomdent
./stop_bot.sh
```

### Перезапуск (после любых изменений)
```bash
cd /Users/macbook/myai/stomdent
./stop_bot.sh && ./start_bot.sh
```

## Внесение изменений

### Изменение базы знаний (цен, FAQ, рекомендаций)
1. Отредактируйте нужный `.md` в `knowledge/`.
2. Перезапустите бота: `./stop_bot.sh && ./start_bot.sh`
3. Проверьте логи на наличие ошибок загрузки: `grep -i "warning\|error" logs/bot.log | tail -20`

### Изменение системного промпта
1. Редактируйте `stdn.txt`.
2. Перезапустите бота.
3. Проверьте, что бот загрузил промпт: `grep "System prompt loaded" logs/bot.log`

### Изменение кода бота (`bot.py`)
- Файл разделён на секции: Logging → Configuration → System prompt/knowledge → History → LLM clients → Handlers → Main.
- Сохраняйте эту структуру и стиль.
- Все текстовые ответы пользователю — на русском или на языке запроса.
- Контактные данные (телефон, адрес) дублируются в `stdn.txt` и в константах `bot.py` — меняйте **в обоих местах** при обновлении.

### Добавление новой команды
1. Добавьте `async def cmd_xxx(update, context)` в секцию Telegram handlers.
2. Зарегистрируйте её в `main()` через `app.add_handler(CommandHandler("xxx", cmd_xxx))`.
3. Добавьте описание команды в `HELP_TEXT` и в `README.md` в таблицу команд.
4. Перезапустите бота.

## Обновление README.md

README — это лицо проекта. При любых значимых изменениях обновляйте его:

- **Новые команды** — добавьте в таблицу "Команды бота".
- **Новые файлы/папки** — обновите блок "Структура проекта".
- **Новые зависимости** — отразите в `requirements.txt` и упомяните в README.
- **Изменения в настройке** — обновите раздел "Настройка".
- **Новые LLM-провайдеры** — добавьте инструкцию в "Необходимые API-ключи".

## Проверка ошибок

### Что делать, если бот не запускается
1. Читайте `logs/bot.log` — там будет конкретная ошибка.
2. Типичные проблемы:
   - `TELEGRAM_BOT_TOKEN не найден` — `.env` отсутствует или некорректен.
   - `Не удалось подключиться к Ollama` — `ollama serve` не запущен.
   - `Модель 'xxx' не найдена в Ollama` — нужно `ollama pull xxx`.
   - `ANTHROPIC_API_KEY не задан` — ключ не вставлен в `.env`.

### Проверка целостности файлов
```bash
cd /Users/macbook/myai/stomdent
python3 -m py_compile bot.py        # Синтаксис Python
python3 -c "from pathlib import Path; [Path('knowledge', f).read_text() for f in ['FAQ.md','prices.md','postop.md']]"  # Markdown читаются
```

### Проверка зависимостей
```bash
cd /Users/macbook/myai/stomdent
source venv/bin/activate
python3 -c "import telegram, dotenv, requests; print('OK')"
```

## Подготовка к публикации

1. **Убедитесь, что `.env` в `.gitignore`** (или не создан `.git` репозиторий).
2. **Проверьте `README.md`** — актуальность структуры, команд, ссылок.
3. **Проверьте `requirements.txt`** — все ли нужные пакеты указаны с версиями.
4. **Проверьте скрипты** на исполняемость:
   ```bash
   chmod +x start_bot.sh stop_bot.sh
   ```
5. **Протестируйте запуск** с нуля (если возможно):
   ```bash
   ./stop_bot.sh
   rm -rf venv bot.pid
   ./start_bot.sh
   ```
6. **Проверьте логи** на отсутствие Warning-ов при старте.
7. **Проверьте Telegram** — отправьте `/start`, `/help`, `/prices` боту.
