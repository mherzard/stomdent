# Работа с проектом STOM Dental Bot

Этот репозиторий — Telegram-бот ассистент стоматологической клиники «STOM» (г. Гавар, Армения).

## Быстрый старт

```bash
cd /Users/macbook/myai/stomdent
./start_bot.sh        # создаст venv, установит зависимости, запустит в фоне
```

Проверка: `tail -f logs/bot.log`
Остановка: `./stop_bot.sh`

## Архитектура

- **bot.py** — единственный файл приложения (Python 3, python-telegram-bot >=20).
- **stdn.txt** — системный промпт для LLM.
- **knowledge/*.md** — база знаний клиники (FAQ, цены, рекомендации).
- **.env** — API-ключи (НЕ коммитить).
- **start_bot.sh / stop_bot.sh** — управление процессом через nohup + PID.

## LLM-провайдеры

Бот поддерживает три провайдера:
- **ollama** (локально, по умолчанию) — требует запущенный `ollama serve`
- **anthropic** — требует `ANTHROPIC_API_KEY`
- **openai** — требует `OPENAI_API_KEY`

Переключается через `LLM_PROVIDER` в `.env`.

## Правила внесения изменений

1. **Все правки — из `/Users/macbook/myai/stomdent`.**
2. **База знаний в `knowledge/`** — единственный источник для LLM. Любые изменения цен/услуг/ответов — только там.
3. **После правок в `knowledge/` или `stdn.txt` бот нужно перезапустить** — файлы читаются на старте.
4. **При изменении кода сохраняйте структуру `bot.py`** — секции разделены комментариями `-----------`.
5. **Обновляйте README.md** при изменении структуры, команд или зависимостей.
6. **Никогда не коммитьте `.env`.**

## Диагностика

- `logs/bot.log` — основной источник ошибок.
- Типичные ошибки:
  - `TELEGRAM_BOT_TOKEN не найден` — нет `.env`
  - `Не удалось подключиться к Ollama` — не запущен `ollama serve`
  - API-ключ начинается с `YOUR_` — ключ не вставлен

## Публикация

Перед публикацией убедитесь:
- `.env` не попадёт в репозиторий
- `README.md` актуален
- `requirements.txt` полный
- `./start_bot.sh` работает с чистого состояния
- команды `/start`, `/help`, `/prices` отвечают в Telegram
